## AI生成图片的后期处理（重命名 → PNG转换）

> ⚠️ **重要**：AI 生成的图片通常为 `.jpg` 格式且文件名为随机哈希，
> 必须经过以下两个 Python 工具处理后才能符合项目 PNG-24 规范。
> 
> 💡 两个工具均支持**两种使用途径**：交互式菜单（无参数运行）和命令行参数。

### 步骤一：批量重命名

使用 `rename_images_by_mtime.py` 按修改时间排序并重命名为规范文件名：

```bash
cd frontend/scripts/

# 方式一：交互式菜单（推荐，无参数运行）
python rename_images_by_mtime.py

# 方式二：命令行参数
# 列出所有子目录及文件统计
python rename_images_by_mtime.py --list

# 按子目录逐批处理（推荐，避免跨子目录命名错位）
python rename_images_by_mtime.py --subdir "基础作物" --preview
python rename_images_by_mtime.py --subdir "基础作物" --execute

# 或全量预览
python rename_images_by_mtime.py --preview

# 确认执行
python rename_images_by_mtime.py --execute
```

### 步骤二：格式转换

使用 `convert_images.py` 将 .jpg 批量转换为 PNG-24：

```bash
cd frontend/scripts/

# 方式一：交互式菜单（推荐，无参数运行）
python convert_images.py

# 方式二：命令行参数
# 转换为 PNG-24 格式（项目规范要求）
python convert_images.py -i ../public/assets/ -o ../public/assets/ -f png
```

### 完整处理流程

```
AI 生成（.jpg + 随机名）
    │
    ▼  rename_images_by_mtime.py
按修改时间排序 → 规范文件名
    │
    ▼  convert_images.py
.jpg → PNG-24 格式转换
    │
    ▼
部署到 public/assets/ ✅
```

## 二、文件命名规范

### 2.1 作物生长阶段图片

| 规范项 | 说明 |
|--------|------|
| 格式 | `{crop_id}_stage_{stage_number}.png` |
| 示例 | `1_stage_1.png` (小麦-幼苗期), `2_stage_3.png` (玉米-开花期) |
| crop_id | 1-84，对应作物ID |
| stage_number | 1-5，对应五个生长阶段 |

**阶段说明：**

| 阶段编号 | 阶段名称 | 说明 | 进度占比 |
|----------|----------|------|----------|
| stage_1 | 幼苗期 | 种子刚发芽，小嫩芽破土 | 0%-20% |
| stage_2 | 生长期 | 茎叶快速生长，植株成形 | 20%-40% |
| stage_3 | 开花期 | 花朵绽放（结果类作物）或叶片繁茂（叶菜类） | 40%-60% |
| stage_4 | 结果期 | 果实初现，逐渐膨大 | 60%-80% |
| stage_5 | 成熟期 | 果实/植株完全成熟，可收获 | 80%-100% |

### 2.2 作物/种子/道具图标

| 图标类型 | 命名格式 | 示例 |
|----------|----------|------|
| 作物图标 | `crop_{crop_id}.png` | `crop_1.png` |
| 种子图标 | `seed_{crop_id}.png` | `seed_2.png` |
| 道具图标 | `item_{item_id}.png` | `item_5.png` |

### 2.3 土地类型图标

| 规范项 | 说明 |
|--------|------|
| 格式 | `land_{quality_id}.png` |
| 示例 | `land_1.png` (普通土地), `land_8.png` (无尽土地) |

### 2.4 UI元素

| 规范项 | 说明 |
|--------|------|
| 格式 | `ui_{category}_{name}.png` |
| 示例 | `ui_button_plant.png`, `ui_panel_bg.png` |
