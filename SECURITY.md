## 报告安全漏洞

如果您发现了安全漏洞，请**不要**在公开 Issue 中报告。请通过以下方式私下报告：

1. 发送邮件至项目维护者（见项目主页）
2. 在报告中包含：漏洞描述、复现步骤、影响范围、建议修复方案

我们承诺在 48 小时内确认收到报告，并在 7 个工作日内提供初始评估和修复计划。

## 安全最佳实践

### 环境变量
- 所有密钥（JWT_SECRET、JWT_ACCESS_SECRET、JWT_REFRESH_SECRET）必须使用强随机字符串（≥32字符）
- 生产环境必须修改所有默认密钥
- `.env` 文件不得提交到版本控制系统

### 认证与授权
- 所有 API 端点必须在 `authMiddleware` 之后注册
- Token 类型必须严格区分（access_token vs refresh_token）
- 刷新令牌轮换时必须将旧令牌加入黑名单
- 密码重置后必须撤销所有活跃令牌

### 数据库安全
- 使用参数化查询防止 SQL 注入
- 数据库连接字符串不得包含特殊字符
- 备份文件应加密存储

### Web 安全
- 所有用户输入必须经过验证和清理
- 启用 CORS 白名单
- 设置适当的安全头（Helmet）
- 实施速率限制防止暴力破解
