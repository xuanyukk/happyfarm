# 开心农场系统测试指南

## 📋 测试前准备

### 1. 配置数据库连接

#### 步骤 1：创建 .env 文件
在 `backend/` 目录下创建 `.env` 文件（复制 `.env.example` 并重命名）：

```bash
cd backend
copy .env.example .env
```

#### 步骤 2：配置数据库连接
编辑 `.env` 文件，修改数据库连接字符串：

```env
# PostgreSQL数据库连接字符串
DATABASE_URL="postgresql://用户名:密码@localhost:5432/happy_farm"

# 例如（使用默认配置）：
# DATABASE_URL="postgresql://postgres:your_db_password@localhost:5432/happy_farm"

# JWT认证配置
JWT_SECRET="your_strong_jwt_secret_key_here"

# 服务器配置
PORT=3001
HOST="localhost"
NODE_ENV="development"

# 安全配置
CORS_ORIGIN="http://localhost:5173,http://localhost:5174"
```

**注意**：请将 `用户名`、`密码` 替换为您实际的PostgreSQL数据库凭据。

---

### 2. 初始化数据库

如果数据库尚未初始化，运行初始化脚本：

```bash
cd sql_init
node init_db.js
```

或者，如果数据库已存在但需要应用新迁移：

#### 应用迁移脚本

**方法一：使用单独的迁移脚本**

```bash
# 先应用阶段一的迁移
cd sql_init/migrations
node migrate_crop_yield_exp.js

# 再应用阶段四的迁移
node migrate_game_activity_log.js
```

**方法二：直接执行SQL迁移脚本**

```sql
-- 在psql或pgAdmin中执行：
-- 009_add_crop_yield_range_and_exp_fields.sql
-- 010_create_game_activity_log_table.sql
```

## 🚀 启动服务

### 3. 启动后端服务

打开终端1，进入backend目录：

```bash
cd backend
npm install  # 如果还没安装依赖
npm run dev
```

**预期输出**：
```
后端服务运行在: http://localhost:3001
数据库连接成功
```

### 4. 启动前端服务

打开终端2，进入frontend目录：

```bash
cd frontend
npm install  # 如果还没安装依赖
npm run dev
```

**预期输出**：
```
VITE vx.x.x  ready in xxx ms

➜  Local:   http://localhost:5173/
➜  Network: use --host to expose
```

## 🐛 问题排查

### 常见问题

**问题1：后端启动失败，提示数据库连接错误**
- 检查 .env 文件中的 DATABASE_URL 是否正确
- 确认 PostgreSQL 服务正在运行
- 确认数据库名称、用户名、密码正确

**问题2：前端启动失败，提示端口被占用**
- 修改 vite.config.js 或使用其他端口
- 或者关闭占用 5173 端口的其他程序

**问题3：活动日志不显示**
- 检查 game_activity_log 表是否已创建
- 检查浏览器控制台是否有错误
- 确认后端服务正常运行，API接口可访问

**问题4：操作后日志不刷新**
- 检查关键操作中是否正确调用了 activityLogRef.value?.refresh()
- 查看浏览器网络请求，确认 API 调用成功
