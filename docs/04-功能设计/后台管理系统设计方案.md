## 1. 项目概述

### 1.1 项目背景

开心农场是一款基于Node.js + Vue 3 + PostgreSQL的农场模拟游戏。后台管理系统作为游戏运营的重要组成部分，负责游戏数据监控、用户管理、系统配置等核心功能。

### 1.2 设计目标

- **安全性：** 完善的权限控制和审计机制
- **可扩展性：** 模块化设计，易于功能扩展
- **可维护性：** 清晰的代码结构和完整文档
- **用户体验：** 直观的操作界面和高效的工作流程
- **数据可视化：** 丰富的图表展示和数据洞察

### 3.2 分层设计原则

#### 3.2.1 前端分层

1. **视图层（View）**
   - 页面组件
   - 布局组件
   - 通用UI组件

2. **状态层（State）**
   - Pinia Store
   - 响应式数据
   - 计算属性

3. **服务层（Service）**
   - API调用
   - 数据转换
   - 错误处理

#### 3.2.2 后端分层

1. **路由层（Routes）**
   - URL路由配置
   - 请求分发
   - 参数验证

2. **控制层（Controllers）**
   - 业务流程编排
   - 响应格式化
   - 错误处理

3. **服务层（Services）**
   - 核心业务逻辑
   - 数据操作
   - 第三方服务集成

4. **中间件层（Middleware）**
   - 认证授权
   - 日志记录
   - 缓存处理
   - 请求验证

### 5.3 权限控制实现

#### 5.3.1 后端权限控制

```javascript
// middleware/permission.js
const checkPermission = (requiredPermission) => {
  return (req, res, next) => {
    const userPermissions = req.user.permissions || [];
    if (!userPermissions.includes(requiredPermission)) {
      return res.status(403).json({
        success: false,
        message: '权限不足'
      });
    }
    next();
  };
};

// routes/adminRoutes.js
router.get('/players', checkAdminPermission, checkPermission('player:view'), getPlayerList);
router.put('/players/:id/status', checkAdminPermission, checkPermission('player:edit'), updatePlayerStatus);
```

#### 5.3.2 前端权限控制

```javascript
// stores/permission.js
export const usePermissionStore = defineStore('permission', () => {
  const permissions = ref([]);
  
  const hasPermission = (requiredPermission) => {
    return permissions.value.includes(requiredPermission);
  };
  
  const hasAnyPermission = (requiredPermissions) => {
    return requiredPermissions.some(p => permissions.value.includes(p));
  };
  
  return {
    permissions,
    hasPermission,
    hasAnyPermission
  };
});
```

#### 14.2.3 技术实现要点

**后端实现**：
```javascript
// 权限验证中间件
const checkPermission = (requiredPermission) => {
    return (req, res, next) => {
        const userPermissions = req.user.permissions || [];
        if (!userPermissions.includes(requiredPermission)) {
            return res.status(403).json({
                success: false,
```

**前端实现**：
```javascript
// 权限指令
export default {
    mounted(el, binding) {
        const { value } = binding;
        const adminStore = useAdminStore();
        
        if (value && value instanceof Array && value.length > 0) {
            const hasPermission = value.some(perm => 
                adminStore.hasPermission(perm)
            );
            if (!hasPermission) {
                el.parentNode && el.parentNode.removeChild(el);
            }
        }
    }
}
```

#### 14.5.3 技术实现要点

**异步处理机制**：
```javascript
// 批量操作服务
async function executeBatchOperation(operationData) {
    const batch = await createBatchRecord(operationData);
    setBatchStatus(batch.id, 'PROCESSING', 0);
    
    for (let i = 0; i < operationData.items.length; i++) {
        try {
            await processItem(operationData.items[i], batch);
            incrementBatchSuccess(batch.id);
        } catch (error) {
            recordBatchError(batch.id, i, error);
        }
        
        const progress = Math.round((i + 1) / operationData.items.length * 100);
        updateBatchProgress(batch.id, progress);
    }
    
    setBatchStatus(batch.id, 'COMPLETED', 100);
}
```

### 14.6 功能五：实时预警推送（WebSocket）

#### 14.6.1 功能总览

完善的预警系统，支持自定义规则、WebSocket实时推送、多渠道通知、处理记录。

**预警类型**：
```
系统预警
├── 服务器资源预警
│   ├── CPU使用率 (CPU_USAGE)
│   ├── 内存使用率 (MEMORY_USAGE)
│   └── 磁盘使用率 (DISK_USAGE)
├── 数据库预警
│   ├── 慢查询预警 (SLOW_QUERY)
│   └── 连接数预警 (DB_CONNECTIONS)
└── API性能预警
    ├── 响应时间过长 (API_RESPONSE_TIME)
    └── 错误率过高 (API_ERROR_RATE)
业务预警
├── 经济异常
│   ├── 大额交易预警 (LARGE_TRANSACTION)
│   └── 货币异常波动 (CURRENCY_FLUCTUATION)
└── 玩家行为
    ├── 异常登录 (ABNORMAL_LOGIN)
    └── 外挂特征检测 (CHEAT_DETECTION)
安全预警
├── 攻击检测 (ATTACK_DETECTION)
└── 权限异常 (PERMISSION_ANOMALY)
```
