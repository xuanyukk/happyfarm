## 📂 配置文件清单

## 🐳 Docker 部署配置

### 配置文件来源

- **模板文件**：`.env.docker.example`
- **启动脚本**：`python start.py`（中英双语菜单，自动生成部署日志）或双击 `start.bat`
- **自动创建**：如果 `.env` 不存在，需手动从模板复制

### 部署流程

```powershell
# 方式一：使用启动脚本（推荐）
python start.py

# 方式二：手动配置后部署
copy .env.docker.example .env
# (可选) 编辑 .env，修改密码等配置
docker-compose up -d --build
```

### Docker Compose 服务

| 服务 | 镜像 | 端口 | 说明 |
|------|------|------|------|
| PostgreSQL | postgres:16-alpine | 5433:5432 | 数据库 |
| Redis | redis:7-alpine | 6379:6379 | 缓存 |
| Backend | 本地构建 | 3001:3001 | 后端API |
| Frontend | 本地构建 | 80:80 | 前端应用 |
| Grafana | grafana/grafana:latest | 3002:3000 | 监控面板 |

## 💻 本地开发配置

### 后端环境变量

复制 `backend/.env.example` 为 `backend/.env`

#### 数据库配置
```env
DATABASE_URL="postgresql://用户名:密码@localhost:5432/happy_farm"
DB_HOST=localhost
DB_PORT=5432
DB_USER=postgres
DB_PASSWORD=your_password_here
DB_NAME=happy_farm
```

#### JWT认证配置
```env
JWT_SECRET="your_strong_jwt_secret_key_here"
JWT_EXPIRES_IN="2h"
JWT_REFRESH_EXPIRES_IN="30d"
```

#### 服务器配置
```env
PORT=3001
HOST="localhost"
NODE_ENV="development"
LOG_LEVEL="debug"
```

#### 安全配置
```env
CORS_ORIGIN="http://localhost:5173,http://localhost:5174"
RATE_LIMIT_WINDOW="15m"
RATE_LIMIT_MAX="100"
ENCRYPTION_KEY="your_strong_encryption_key_here"
```

#### Redis配置（可选）
```env
REDIS_ENABLED="false"
REDIS_HOST="localhost"
REDIS_PORT="6379"
REDIS_PASSWORD=""
REDIS_DB="0"
```

#### 邮件配置（可选）
```env
SMTP_ENABLED="false"
SMTP_HOST="smtp.example.com"
SMTP_PORT="587"
SMTP_USER="your-email@example.com"
SMTP_PASS="your-email-password"
SMTP_FROM="no-reply@example.com"
```

### 前端环境变量

复制 `frontend/.env.example` 为 `frontend/.env`

```env
# API接口地址
VITE_API_URL=http://localhost:3001/api

# 日志配置
VITE_LOG_LEVEL=debug
VITE_ENABLE_LOG_REPORT=false
VITE_ENABLE_LOG_CONSOLE=true
```

## 🔧 环境变量详解

### 数据库配置

| 变量名 | 默认值 | 说明 |
|--------|--------|------|
| `POSTGRES_DB` | `happy_farm` | 数据库名称 |
| `POSTGRES_USER` | `postgres` | 数据库用户名 |
| `POSTGRES_PASSWORD` | - | 数据库密码（必须修改） |
| `DB_HOST` | `localhost` | 数据库主机地址 |
| `DB_PORT` | `5432` | 数据库端口 |

### JWT认证配置

| 变量名 | 默认值 | 说明 |
|--------|--------|------|
| `JWT_SECRET` | - | JWT签名密钥（必须设置强密钥） |
| `JWT_EXPIRES_IN` | `2h` | 访问令牌有效期 |
| `JWT_REFRESH_EXPIRES_IN` | `30d` | 刷新令牌有效期 |

### 服务器配置

| 变量名 | 默认值 | 说明 |
|--------|--------|------|
| `PORT` | `3001` | 后端服务端口 |
| `HOST` | `localhost` | 服务监听地址 |
| `NODE_ENV` | `development` | 运行环境 |
| `LOG_LEVEL` | `debug` | 日志输出级别 |

### 安全配置

| 变量名 | 默认值 | 说明 |
|--------|--------|------|
| `CORS_ORIGIN` | `http://localhost:5173` | 允许的跨域来源 |
| `RATE_LIMIT_WINDOW` | `15m` | 速率限制时间窗口 |
| `RATE_LIMIT_MAX` | `100` | 最大请求次数 |
| `ENCRYPTION_KEY` | - | 数据加密密钥 |
| `SAFETY_CONFIRM` | `true` | 是否启用安全确认 |

## 🛡️ 安全最佳实践

1. **不要提交 `.env` 到 Git**
   - 所有 `.env` 文件已在 `.gitignore` 中配置忽略

2. **生产环境必须修改所有默认密码**
   - 数据库密码
   - JWT密钥
   - 加密密钥
   - Redis密码（如启用）

3. **使用强密码**
   - 包含大小写字母、数字和特殊字符
   - 长度至少16位

4. **定期轮换密码和密钥**
   - 建议每3-6个月轮换一次

### 密钥生成命令

```powershell
# 生成JWT密钥（64字节）
node -e "console.log(require('crypto').randomBytes(64).toString('hex'))"

# 生成加密密钥（32字节）
node -e "console.log(require('crypto').randomBytes(32).toString('hex'))"
```
