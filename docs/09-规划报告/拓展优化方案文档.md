**watering_boost 效果计算规则**：
- 基础浇水耗时 = 5秒/地块
- 使用青铜水壶后 = 5 × 0.80 = 4秒/地块（节省20%时间）
- 使用银质水壶后 = 5 × 0.60 = 3秒/地块（节省40%时间）
- 使用金质水壶后 = 5 × 0.40 = 2秒/地块（节省60%时间）
- 效果持续时间到期后自动恢复原始浇水速度
- 多次使用不叠加，新效果覆盖旧效果

**fertilize_boost 效果计算规则**：
- 超级肥料包：施肥增产效果在原基础上 ×1.50
- 例如：高级增产剂(2.0x产量) + 超级肥料包 = 3.0x产量
- 仅对单次施肥操作生效，使用后即消耗

**harvest_boost 效果计算规则**：
- 丰收镰刀：在作物基础产量上额外增加30%
- 例如：作物基础产量100 + 增产剂200% + 镰刀30% = 330（乘法叠加）
- 对单次收获操作生效，使用后消耗

#### 4.3.2 每日限时折扣功能

**折扣机制设计**：

| 属性 | 说明 |
|------|------|
| 折扣数量 | 每日随机抽取3个商品进行限时折扣 |

| 折扣时段 | 每日00:00开始，持续24小时 |
| 刷新规则 | 每日00:00服务器时间刷新，旧折扣清除，新折扣随机生成 |
| 适用范围 | 农场币商店和宝石币商店均可参与 |
| 排除商品 | 种子商品不参与折扣（避免经济崩坏） |

**折扣计算规则**：

```
折扣价格 = 原价 × discount_rate（向下取整）

示例：
  中级增产剂原价 200 金币，discount_rate=0.75
  折扣价格 = 200 × 0.75 = 150 金币

  春日皮肤原价 50 宝石，discount_rate=0.80
  折扣价格 = 50 × 0.80 = 40 宝石
```

**折扣商品抽取规则**：

```
1. 每日00:00执行折扣刷新定时任务
2. 从所有非种子、已解锁、库存>0的商品中随机抽取3个
3. 每个商品的折扣率独立随机（0.70~0.90，步长0.05）
4. 同一商品连续2天不参与打折（冷却机制，防止单一商品频繁打折）
5. 打折商品在商店UI中以橙色标签标记，显示原价（划线）和折扣价
```

**前端倒计时显示规范**：

| 元素 | 说明 |
|------|------|
| 折扣标签 | 橙色背景圆角标签，显示"限时X折" |
| 原价显示 | 灰色文字 + 删除线 |
| 折扣价 | 橙色加粗文字 |
| 倒计时 | HH:MM:SS格式，每秒更新 |
| 倒计时位置 | 折扣商品卡片右下角 |
| 到期效果 | 倒计时归零后商品自动恢复原价，标签消失 |

**BOSS技能机制**：

| 技能类型 | 说明 | 示例 |
|----------|------|------|
| 普通攻击 | 每回合对玩家造成攻击力80%-120%的伤害 | 杂草王普通攻击：16-24伤害 |

**BOSS战战斗流程**：

```
1. 进入阶段：消耗体力，选择出战作物（选3块地的作物转化为"战力值"）
   战力值 = Σ(地块品质系数 × 作物等级系数)
   品质系数：普通1.0→铜1.5→银2.0→金3.0→铂金4.5→钻石6.0→无尽8.0→永恒10.0
   作物等级系数：作物ID范围映射 1-15→1.0, 16-34→2.0, 35-74→4.0, 75-84→8.0

2. 战斗阶段（回合制）：
   - 玩家回合：选择"攻击"/"防御"/"道具"
     * 攻击：造成 战力值 × (1 - BOSS防御力/1000) 伤害
     * 防御：本回合受到伤害减半
     * 道具：使用战斗专用道具（如"除草剂"对杂草王造成额外伤害）
   - BOSS回合：执行技能（普攻/蓄力/特殊技能）

3. 结算阶段：
   - 胜利：获得经验+掉落物品+首通奖励
   - 失败：获得少量经验，扣除的体力返还50%
   - 超时（30回合未击败）：判定失败

4. 奖励掉落规则：
   - 基础掉落：100%掉落金币（BOSS等级×100金币）
   - 概率掉落：60%掉落1件道具，20%掉落2件，5%掉落3件
```

#### 5.3.4 交易系统设计

**可交易物品**：

| 类别 | 可交易 | 不可交易 |
|------|--------|----------|
| 道具(消耗品) | 增产剂、加速剂、体力药水等 | 限时皮肤(已激活)、神秘宝箱(已开启) |
| 道具(永久) | 土地祝福、丰收之神等 | — |
| 作物 | 已收获的作物果实 | 未收获的作物、种子 |
| 装饰物 | 木栅栏、稻草人 | 已放置的装饰物 |
| 货币 | ❌ 不可交易 | 金币、宝石币均不可交易 |

**交易流程**：

```
1. 发起交易
   发起方选择要提供的物品和希望获得的物品
   → 创建交易订单（状态：pending）

2. 确认交易
   接收方收到交易通知
   → 可接受（状态：accepted）或拒绝（状态：rejected）或还价（创建新订单）

3. 安全验证
   系统验证双方背包：
   - 发起方物品是否足够
   - 接收方物品是否足够
   - 物品是否在可交易白名单内
   → 验证通过后执行交换

4. 交易完成
   双方背包物品原子交换
   → 记录交易日志
   → 双方收到交易完成通知

5. 超时处理
   交易订单24小时未确认 → 自动取消
```

**安全验证机制**：

| 验证项 | 说明 |
|--------|------|
| 数量验证 | 交易数量不能超过持有数量 |
| 冷却验证 | 同一对玩家间交易冷却5分钟 |
| 频率限制 | 单玩家每日最多发起20笔交易 |
| 异常检测 | 短时间大量高价值物品交易触发风控标记 |
| 原子操作 | 使用数据库事务确保交换全部成功或全部回滚 |
| 日志审计 | 所有交易记录完整日志，支持回溯查询 |

**交易费用（防止刷物品）**：

| 费用类型 | 说明 |
|------|------|
| 交易税 | 接收方支付交易物品估值的5%作为金币手续费 |
| 最低税费 | 每笔交易最低收取100金币 |
| 税费上限 | 单笔交易最高收取5000金币 |
| 新友保护 | 好友关系<3天的交易，税费翻倍（防小号刷物品） |

**交易数据模型**：

```sql
CREATE TABLE trade_order (
    order_id SERIAL PRIMARY KEY,
    initiator_id VARCHAR(64) NOT NULL,

    status VARCHAR(20) DEFAULT 'pending',  -- pending/accepted/rejected/cancelled/expired
    initiator_offers JSONB NOT NULL,       -- [{item_id, item_name, quantity}]
    receiver_offers JSONB NOT NULL,        -- [{item_id, item_name, quantity}]
    tax_amount INT DEFAULT 0,
    created_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP,
    accepted_at TIMESTAMP,
    expires_at TIMESTAMP DEFAULT (CURRENT_TIMESTAMP + INTERVAL '24 hours')

CREATE TABLE trade_log (

    tax_paid INT DEFAULT 0,
    trade_time TIMESTAMP DEFAULT CURRENT_TIMESTAMP
```
