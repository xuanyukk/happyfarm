# 农场装饰系统详细玩法设计方案

---

## 一、系统概述

### 1.1 设计目标

农场装饰系统是开心农场的核心个性化表达系统，允许玩家通过放置各类装饰元素来美化自己的农场空间。该系统与核心种植玩法并行，提供以下价值：

| 目标 | 说明 |
|------|------|
| 个性化表达 | 每位玩家拥有独一无二的农场外观，展示个人审美 |
| 长期目标 | 装饰物收集和摆放提供超越种植的中长期游戏目标 |
| 社交价值 | 农场美观度成为好友互访的核心看点 |
| 功能增益 | 部分装饰物提供微弱的种植/收获效率加成 |
| 商业化潜力 | 装饰物是宝石币消费的核心场景之一 |

### 1.2 装饰物分类总览

| 类别 | 装饰物示例 | 数量规划 | 获取途径 | 功能增益 | 视觉权重 |
|------|-----------|----------|----------|----------|:--------:|
| 围栏(Fence) | 木栅栏、石墙、花篱、铁栅栏、水晶栏 | 5种×20个=100件 | 商店+活动+成就 | 围合区域作物生长+3% | ⭐ |
| 路面(Path) | 石板路、鹅卵石路、花径、木栈道、彩虹路 | 5种×50段=250段 | 商店+等级解锁 | 相邻地块操作速度+5% | ⭐ |
| 绿植(Plant) | 花丛、灌木、大树、盆栽、藤架 | 10种×10个=100件 | 商店+掉落 | 无(纯装饰) | ⭐⭐ |
| 小屋(Building) | 工具屋、仓库、宠物窝、凉亭、温室 | 5种×1个=5栋 | 世界等级+任务 | 仓库增加存储上限 | ⭐⭐⭐ |
| 景观(Landscape) | 喷泉、风车、雕像、池塘、瞭望塔 | 8种×1个=8个 | 成就+活动+宝石 | 全农场美观度+N | ⭐⭐⭐⭐⭐ |
| 特效(Effect) | 萤火虫群、彩虹、流星雨 | 3种 | 活动限时 | 限时美观+幸运加成 | 动态特效 |

### 2.3 美观度评分系统

美观度是衡量农场装饰水平的核心数值，影响排名和部分社交奖励。

**美观度计算公式**：

```
农场美观度 = Σ(装饰物基础分 × 摆放位置系数 × 组合加成系数)

基础分:
  - 围栏: 2分/个
  - 路面: 3分/段
  - 绿植: 5分/个
  - 小屋: 20分/栋
  - 景观: 50分/个

摆放位置系数:
  - 农场边缘(距边界≤2格): 0.8
  - 农场中央(距边界>2格): 1.0
  - 地块间空隙: 1.2

组合加成:
  - 围栏连续≥8段: ×1.1
  - 路面连续≥5段: ×1.1
  - 小屋+四周绿植≥4个: ×1.2
  - 景观+四周围栏完整: ×1.3
  - 全农场绿化率≥30%: ×1.15
  - 全农场绿化率≥60%: ×1.25
```

## 三、用户交互流程

### 3.1 装饰物购买流程

```
1. 打开装饰商店
   入口：主界面"装饰"按钮 / 商店页"装饰"标签
   → 展示5个分类标签页（围栏/路面/绿植/小屋/景观）
   → 每类显示已拥有/未拥有的装饰物列表
   → 预览功能：点击装饰物可在农场预览放置效果

2. 选择装饰物
   → 查看详情面板：
     - 3D/2D预览图
     - 基础属性（尺寸、美观度分）
     - 价格（金币/宝石币）
     - 解锁条件（等级/成就）
     - 已拥有数量
   → 点击"购买"按钮

3. 确认购买
   → 弹出确认对话框
   → 显示余额变化
   → 购买成功动画（金币飞入背包）
   → 装饰物加入背包
```

### 3.2 装饰物放置流程

```
1. 进入编辑模式
   入口：主界面"编辑"按钮 / 背包中点击装饰物"放置"
   → 农场进入编辑模式：
     - 地块网格线显示
     - 现有装饰物半透显示
     - 可放置区域高亮（绿色）
     - 不可放置区域灰暗（红色）

2. 选择装饰物
   → 从背包/装饰面板选择要放置的物品
   → 物品跟随光标移动
   → 实时碰撞检测

3. 定位与旋转
   → 拖动到目标位置
   → 点击旋转按钮（或按R键）旋转方向
   → 支持4方向(0°/90°/180°/270°)
   → 围栏自动吸附到地块边界

4. 确认放置
   → 点击"确认"或按Enter
   → 扣减背包中的装饰物
   → 装饰物固定到位置
   → 播放放置动画（缩放弹入效果）

5. 撤销/取消
   → 点击"取消"或按ESC
   → 装饰物退回背包
   → 退出编辑模式
```

### 3.3 装饰物移除/调整流程

```
1. 进入编辑模式 → 点击已放置的装饰物
2. 弹出操作菜单：
   - 移动：重新拖放到新位置
   - 旋转：切换4方向
   - 回收：放回背包（不消耗额外资源）
   - 替换：用新装饰物替换（旧装饰退回背包）
3. 移除确认 → 回收动画（缩小消失）
```

### 3.4 围栏连续围合检测

```
检测算法：
1. 玩家放置栅栏后触发检测
2. 从新放置的栅栏出发，沿4方向BFS搜索连通的栅栏
3. 判断连通栅栏是否形成闭合回路：
   - 路径回到起点 → 判定为围合区域
   - 统计围合区域内的地块数量
4. 围合≥8块地块 → 内部地块获得"围合加成"效果（生长速度+3%）
5. 前端围合区域高亮显示（半透明绿色覆盖）
```

### 5.2 主题风格一致性

所有装饰物采用统一的"童话农场"美术风格：

| 风格要素 | 规范 |
|----------|------|
| 主色调 | 暖色系为主(棕、绿、橙)，高饱和度 |
| 线条 | 柔边圆角，描边宽度2px深棕色 |
| 阴影 | 右下45°投影，偏移4px，透明度30% |
| 透视 | 等距45°倾斜视角(Isometric) |
| 比例 | 小屋>景观>绿植>围栏>路面，保持现实比例 |

### 5.3 渲染优先级与遮挡规则

```
等距排序算法（核心）：
  对于所有可见装饰物，按以下规则排序后依次渲染：

  规则1(主要): Y坐标大的先渲染 → 模拟"前方遮挡后方"
  规则2(次要): 同Y坐标时，X坐标小的先渲染
  规则3(层级): 围栏(Fence层) > 路面(Path层) > 绿植(Plant层)
                > 建筑后层(Building_Back) > 作物层(Crop层)
                > 建筑前层(Building_Front) > 景观(Landscape层)

  排序伪代码：
  decorations.sort((a, b) => {
    if (a.layerPriority !== b.layerPriority)
      return a.layerPriority - b.layerPriority
    if (a.gridY !== b.gridY)
      return a.gridY - b.gridY
    return a.gridX - b.gridX
  })
```

## 七、数据库架构设计

### 7.1 装饰物配置表

```sql
CREATE TABLE decoration_config (
    decoration_id SERIAL PRIMARY KEY,
    category VARCHAR(20) NOT NULL,       -- fence/path/plant/building/landscape/effect
    deco_name VARCHAR(50) NOT NULL,
    deco_desc VARCHAR(200),
    sprite_path VARCHAR(200) NOT NULL,   -- 资源路径
    width_grid INT NOT NULL DEFAULT 1,   -- 占用地块宽度
    height_grid INT NOT NULL DEFAULT 1,  -- 占用地块高度
    beauty_score INT NOT NULL DEFAULT 0, -- 美观度基础分
    func_boost JSONB,                    -- 功能增益 {"boost_type": "growth_speed", "value": 0.03}
    buy_price_gold INT DEFAULT 0,        -- 金币价格（0=不可用金币购买）
    buy_price_gem INT DEFAULT 0,         -- 宝石价格（0=不可用宝石购买）
    unlock_world_level INT DEFAULT 1,    -- 解锁世界等级
    unlock_player_level INT DEFAULT 1,   -- 解锁玩家等级
    max_place_count INT DEFAULT 999,     -- 最大放置数量
    is_limited BOOLEAN DEFAULT FALSE,    -- 是否限量
    weekly_stock INT DEFAULT 0,          -- 每周库存（0=不限量）
    is_active BOOLEAN DEFAULT TRUE
);
```

## 八、前端技术实现方案

### 8.1 渲染方案选择

| 方案 | 适用场景 | 优点 | 缺点 |
|------|----------|------|------|
| Canvas 2D | 全部装饰物渲染 | 性能好(单画布)，支持等距排序 | 需要自行实现碰撞和事件 |
| DOM + CSS | 简单装饰物(围栏/路面) | 事件处理简单 | 大量元素时性能差 |
| WebGL(PixiJS) | 高复杂度场景(景观+粒子) | 性能最优，支持shader | 学习成本高 |

**推荐方案**：PixiJS（轻量WebGL框架）作为主渲染引擎，DOM用于UI交互层。

### 8.3 性能优化策略

| 策略 | 实现方式 |
|------|----------|
| 视口裁剪 | 仅渲染屏幕可见区域内的装饰物（超出屏幕±200px不渲染） |
| 对象池 | 围栏/路面等同类型装饰物使用同一纹理的不同实例 |
| 脏标记 | 仅当装饰物位置/状态变化时重新排序和渲染 |
| 纹理图集 | 所有装饰物打包为2-3个Sprite Atlas纹理图集 |
| 批量渲染 | 同纹理的装饰物合并为单次Draw Call |
| LOD切换 | 缩放<0.5时使用低精度纹理(32px)，缩放在0.5-1.0时使用中精度(64px)，缩放>1.0使用高精度(128px) |
