# 日志分析平台配置指南

## 一、概述

本项目已配置Winston日志系统，支持多种日志输出方式。为了实现完整的日志分析功能，可以集成以下平台之一。

## 二、ELK Stack (Elasticsearch + Logstash + Kibana)

### 2.1 架构说明
- **Elasticsearch**: 搜索引擎，存储和索引日志数据
- **Logstash**: 日志收集和处理管道
- **Kibana**: 可视化界面，用于日志分析和监控

### 2.2 Docker快速部署

```yaml
# docker-compose.yml
version: '3'
services:
  elasticsearch:
    image: docker.elastic.co/elasticsearch/elasticsearch:8.11.0
    environment:
      - discovery.type=single-node
      - xpack.security.enabled=false
    ports:
      - "9200:9200"
    volumes:
      - esdata:/usr/share/elasticsearch/data

  logstash:
    image: docker.elastic.co/logstash/logstash:8.11.0
    volumes:
      - ./logstash.conf:/usr/share/logstash/pipeline/logstash.conf
      - ./backend/logs:/usr/share/logstash/logs
    depends_on:
      - elasticsearch

  kibana:
    image: docker.elastic.co/kibana/kibana:8.11.0
    ports:
      - "5601:5601"
    depends_on:
      - elasticsearch

volumes:
  esdata:
```

### 2.3 Logstash配置

```ruby
# logstash.conf
input {
  file {
    path => "/usr/share/logstash/logs/*.log"
    start_position => "beginning"
    sincedb_path => "/dev/null"
  }
}

filter {
  json {
    source => "message"
  }
}

output {
  elasticsearch {
    hosts => ["elasticsearch:9200"]
    index => "happy-farm-logs-%{+YYYY.MM.dd}"
  }
}
```

## 三、Grafana + Loki

### 3.1 架构说明
- **Loki**: 轻量级日志聚合系统
- **Grafana**: 可视化界面，支持日志查询和仪表板

### 3.2 Docker快速部署

```yaml
# docker-compose.yml
version: '3'
services:
  loki:
    image: grafana/loki:2.9.0
    ports:
      - "3100:3100"
    command: -config.file=/etc/loki/local-config.yaml

  promtail:
    image: grafana/promtail:2.9.0
    volumes:
      - ./backend/logs:/var/log
      - ./promtail-config.yml:/etc/promtail/config.yml
    command: -config.file=/etc/promtail/config.yml

  grafana:
    image: grafana/grafana:10.2.0
    ports:
      - "3002:3000"
    environment:
      - GF_SECURITY_ADMIN_USER=admin
      - GF_SECURITY_ADMIN_PASSWORD=happyfarm123
```

### 3.3 Promtail配置

```yaml
# promtail-config.yml
server:
  http_listen_port: 9080

positions:
  filename: /tmp/positions.yaml

clients:
  - url: http://loki:3100/loki/api/v1/push

scrape_configs:
  - job_name: happy-farm-logs
    static_configs:
      - targets:
          - localhost
        labels:
          job: happy-farm
          __path__: /var/log/*.log
```

## 四、日志格式

本项目的Winston日志格式：

```json
{
  "timestamp": "2026-03-19T12:00:00.000Z",
  "level": "info",
  "message": "用户登录成功",
  "metadata": {
    "userId": 1,
    "ip": "127.0.0.1"
  }
}
```

## 五、使用说明

1. **启动服务**: 使用 `docker-compose up -d` 启动相应的日志分析平台
2. **访问界面**: 
   - Kibana: http://localhost:5601
   - Grafana: http://localhost:3002
3. **配置数据源**: 在界面中添加相应的数据源（Elasticsearch或Loki）
4. **创建仪表板**: 根据需要创建日志监控仪表板

## 六、推荐配置

对于小型项目，推荐使用 **Grafana + Loki**，因为：
- 部署更简单
- 资源占用更少
- 配置更便捷

对于大型项目，推荐使用 **ELK Stack**，因为：
- 功能更强大
- 搜索性能更好
- 生态系统更完善
