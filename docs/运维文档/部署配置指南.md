## 🌍 环境要求

### 最低要求
- **Node.js**: 18.x 或更高版本
- **PostgreSQL**: 14.x 或更高版本
- **Redis**: 7.x 或更高版本（推荐）
- **npm**: 8.x 或更高版本（或 pnpm 7.x+）
- **操作系统**: Windows 10+ / macOS 10.15+ / Linux (Ubuntu 20.04+)

### 推荐配置
- **Node.js**: 18.x LTS
- **PostgreSQL**: 16.x
- **Redis**: 7.x
- **内存**: 8GB+
- **磁盘**: 10GB+ 可用空间

## 🐳 Docker部署配置

### 快速部署

```powershell
# 方式一：使用 Python 启动脚本（推荐）
python start.py

# 方式二：手动配置后部署
copy .env.docker.example .env
# (可选) 编辑 .env，修改密码等配置
docker-compose up -d --build
```

### Docker Compose 服务配置

项目使用 `docker-compose.yml` 配置以下服务：

| 服务 | 镜像 | 端口映射 | 说明 |
|------|------|---------|------|
| PostgreSQL | postgres:16-alpine | 5433:5432 | 数据库服务 |
| Redis | redis:7-alpine | 6379:6379 | 缓存服务 |
| Backend | 本地构建 | 3001:3001 | 后端API服务 |
| Frontend | 本地构建 | 80:80 | 前端应用 |
| Grafana | grafana/grafana:latest | 3002:3000 | 监控面板 |

## 💻 本地开发配置

### 后端配置

复制 `backend/.env.example` 为 `backend/.env`，主要配置项：

```env
# 数据库连接
DATABASE_URL="postgresql://用户名:密码@localhost:5432/happy_farm"
DB_HOST=localhost
DB_PORT=5432
DB_USER=postgres
DB_PASSWORD=your_password_here
DB_NAME=happy_farm

# JWT认证
JWT_SECRET="your_strong_jwt_secret_key_here"
JWT_EXPIRES_IN="2h"
JWT_REFRESH_EXPIRES_IN="30d"

# 服务器配置
PORT=3001
HOST="localhost"
NODE_ENV="development"

# 日志配置
LOG_LEVEL="debug"

# CORS配置
CORS_ORIGIN="http://localhost:5173,http://localhost:5174"

# 安全配置
RATE_LIMIT_WINDOW="15m"
RATE_LIMIT_MAX="100"

# Redis配置（可选）
REDIS_ENABLED="false"
REDIS_HOST="localhost"
REDIS_PORT="6379"

# 邮件配置（可选）
SMTP_ENABLED="false"
SMTP_HOST="smtp.example.com"
SMTP_PORT="587"
SMTP_USER="your-email@example.com"
SMTP_PASS="your-email-password"
```

### 前端配置

复制 `frontend/.env.example` 为 `frontend/.env`，主要配置项：

```env
# API接口地址
VITE_API_URL=http://localhost:3001/api

# 日志配置
VITE_LOG_LEVEL=debug
VITE_ENABLE_LOG_REPORT=false
VITE_ENABLE_LOG_CONSOLE=true
```

## 🔧 环境变量详解

### 数据库配置

| 变量名 | 默认值 | 说明 |
|--------|--------|------|
| `POSTGRES_DB` | `happy_farm` | 数据库名称 |
| `POSTGRES_USER` | `postgres` | 数据库用户名 |
| `POSTGRES_PASSWORD` | - | 数据库密码（必须修改） |
| `DB_HOST` | `localhost` | 数据库主机地址 |
| `DB_PORT` | `5432` | 数据库端口 |

### JWT认证配置

| 变量名 | 默认值 | 说明 |
|--------|--------|------|
| `JWT_SECRET` | - | JWT签名密钥（必须设置强密钥） |
| `JWT_EXPIRES_IN` | `2h` | 访问令牌有效期 |
| `JWT_REFRESH_EXPIRES_IN` | `30d` | 刷新令牌有效期 |

### 服务器配置

| 变量名 | 默认值 | 说明 |
|--------|--------|------|
| `PORT` | `3001` | 后端服务端口 |
| `HOST` | `localhost` | 服务监听地址 |
| `NODE_ENV` | `development` | 运行环境 |

### 安全配置

| 变量名 | 默认值 | 说明 |
|--------|--------|------|
| `CORS_ORIGIN` | `http://localhost:5173` | 允许的跨域来源 |
| `RATE_LIMIT_WINDOW` | `15m` | 速率限制时间窗口 |
| `RATE_LIMIT_MAX` | `100` | 最大请求次数 |
| `ENCRYPTION_KEY` | - | 数据加密密钥 |

## 🗄️ 数据库配置

### 初始化数据库

```powershell
# 1. 创建数据库
psql -U postgres -c "CREATE DATABASE happy_farm;"

# 2. 执行初始化脚本
cd sql_init
node init_db.js
```

### 数据库迁移

```powershell
cd backend
npm run migrate
npm run seed
```

## 🛡️ 安全配置

### 生产环境必做事项

1. **修改所有默认密码**
   - 数据库密码
   - JWT密钥
   - 加密密钥

2. **启用HTTPS**
   - 配置SSL证书
   - 强制HTTPS重定向

3. **配置防火墙**
   - 仅开放必要端口
   - 限制数据库访问

4. **启用Redis**
   - 设置Redis密码
   - 配置持久化

### 密钥生成命令

```powershell
# 生成JWT密钥
node -e "console.log(require('crypto').randomBytes(64).toString('hex'))"

# 生成加密密钥
node -e "console.log(require('crypto').randomBytes(32).toString('hex'))"
```

## ❓ 常见问题

### Q: 端口被占用怎么办？

```powershell
# 查找占用端口的进程
netstat -ano | findstr ":3001"

# 结束进程（替换PID）
taskkill /PID <进程ID> /F
```

### Q: 数据库连接失败？

1. 确认PostgreSQL服务正在运行
2. 检查 `.env` 文件中的数据库配置
3. 验证用户名和密码

### Q: 如何修改已部署的配置？

```powershell
# 1. 编辑 .env
notepad .env

# 2. 重启相关服务
docker-compose restart backend postgres redis
```
