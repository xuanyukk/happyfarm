# 代码审查检查清单

本文档提供开心农场项目的代码审查检查清单，确保代码质量、安全性和可维护性。

---

## PR准备检查

提交PR前，作者应确认：

- [ ] 功能已完整实现
- [ ] 代码已格式化
- [ ] 运行ESLint检查通过
- [ ] 运行单元测试通过
- [ ] 文档已更新（如需要）
- [ ] PR描述清晰说明变更内容

---

## 代码审查检查清单

### 1. 功能性检查

- [ ] 功能是否完整实现需求？
- [ ] 是否处理了边界情况？
- [ ] 是否处理了错误情况？
- [ ] 是否存在逻辑bug？

### 2. 代码质量检查

- [ ] 代码可读性良好？
- [ ] 命名规范？
- [ ] 函数/方法长度适中？
- [ ] 是否有重复代码？
- [ ] 是否遵循项目编码规范？

### 3. 安全性检查

- [ ] 是否存在SQL注入风险？
- [ ] 是否存在XSS风险？
- [ ] 输入是否验证和转义？
- [ ] 认证和授权是否正确？
- [ ] 敏感数据是否安全存储？
- [ ] 是否存在CSRF风险？
- [ ] 是否有安全配置问题？

### 4. 性能检查

- [ ] 是否存在性能瓶颈？
- [ ] 数据库查询是否优化？
- [ ] 是否有N+1查询问题？
- [ ] 缓存是否合理使用？
- [ ] 前端渲染是否优化？
- [ ] 是否有不必要的计算？

### 5. 测试检查

- [ ] 新功能是否有测试？
- [ ] 测试覆盖是否足够？
- [ ] 测试是否可以独立运行？
- [ ] 测试断言是否合理？
- [ ] 是否有集成测试？

### 6. 文档检查

- [ ] 代码注释是否充分？
- [ ] API文档是否更新？
- [ ] 用户文档是否需要更新？
- [ ] README是否需要更新？
- [ ] 变更记录是否更新？

### 7. 架构与设计检查

- [ ] 代码是否符合架构设计？
- [ ] 依赖关系是否合理？
- [ ] 是否引入了不必要的依赖？
- [ ] 是否考虑了可扩展性？
- [ ] 模块划分是否清晰？

### 8. 前端特定检查

- [ ] 组件设计是否合理？
- [ ] 状态管理是否恰当？
- [ ] 用户体验是否良好？
- [ ] 响应式设计是否处理？
- [ ] 加载和错误状态是否处理？
- [ ] 性能是否优化？

### 9. 后端特定检查

- [ ] API设计是否RESTful？
- [ ] 错误响应格式是否统一？
- [ ] 日志记录是否充分？
- [ ] 事务处理是否正确？
- [ ] 资源清理是否完整？

### 10. 可维护性检查

- [ ] 代码是否容易理解？
- [ ] 是否容易后续修改？
- [ ] 是否遵循SOLID原则？
- [ ] 技术债务是否可控？
- [ ] 警告信息是否处理？

---

## 审查评级

### ✅ 通过
- 所有检查点满足
- 代码质量良好
- 可以合并

### ⚠️ 有条件通过
- 有一些小问题
- 需要轻微修改
- 修改后可合并

### ❌ 拒绝
- 存在严重问题
- 需要较大修改
- 重新提交审查

---

## 审查指南

### 积极正面的反馈
- 肯定好的实践
- 给予鼓励和建议

### 具体明确的评论
- 指出具体位置
- 提供具体建议
- 解释为什么需要修改

### 尊重不同观点
- 不同方案可能各有优劣
- 充分讨论，达成共识

## 常见审查问题示例

### 安全性问题
```
问题：这里的SQL查询使用字符串拼接，存在SQL注入风险
建议：使用参数化查询
```

### 性能问题
```
问题：这个循环中有数据库查询，会造成N+1问题
建议：提前批量查询
```

### 可维护性问题
```
问题：这个函数过长，可以拆分
建议：拆分成更小的函数
```

## 审查后的行动

### 需要修改时
1. 作者根据评论修改代码
2. 推送到同一分支
3. 请求重新审查

### 已批准时
1. 合并到目标分支
2. （可选）删除功能分支
3. 关闭相关Issue（如有）

## 安全加固专项审查

### XSS防护审查
- [ ] 所有用户输入是否进行了适当的转义处理？
- [ ] 是否使用了安全的HTML渲染方式？
- [ ] 是否对危险的payload进行了检测？
- [ ] 是否对属性值进行了清理？
- [ ] 前端组件是否正确处理了用户生成内容？

### SQL注入防护审查
- [ ] 是否使用了参数化查询？
- [ ] 是否对输入进行了类型验证？
- [ ] 是否避免了字符串拼接SQL？
- [ ] 是否使用了ORM的安全特性？
- [ ] 是否有输入长度限制？

### 权限边界测试审查
- [ ] 超级管理员权限是否正确实现？
- [ ] 角色权限是否正确验证？
- [ ] 资源所有权检查是否到位？
- [ ] 越权访问是否被阻止？
- [ ] 权限提升是否被防护？
