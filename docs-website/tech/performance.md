# 性能优化

本文档介绍开心农场项目的性能优化方案和最佳实践。

## 优化目标

- 提高系统响应速度
- 降低资源消耗
- 提升用户体验
- 支持更多并发用户

## 数据库优化

> **v4.60.1 更新**：已完成全面的数据库性能优化。

### 1. 索引优化
- 为常用查询字段添加索引
- 使用复合索引
- 定期分析索引使用情况
- **v4.60.1 已补全**：3个高频查询缺失复合索引（crop_unlock、shop_goods_unlock、currency_log）

### 2. 查询优化
- 避免 N+1 查询
- 使用连接查询（LEFT JOIN / UNION ALL）
- 分页查询
- 查询缓存
- **v4.60.1 已修复**：`economyService` N+1 查询、`cropService`/`shopService`/`playerService` 中 SELECT * 替换为精确字段

### 3. 事务优化
- 单事务批量提交代替逐行事务
- 批量 UPDATE 代替循环 UPDATE
- 连接池配置优化
- **v4.60.1 已修复**：`cropService` 逐行事务改为单事务、`itemService` 循环 UPDATE 改为批量操作

## 性能基准

### 响应时间目标
- API 响应 < 100ms
- 页面加载 < 2s
- 首屏渲染 < 1s

### 并发支持
- 支持 1000+ 并发用户
- 数据库连接池 100+
- Redis 连接池 50+
