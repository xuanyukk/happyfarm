# 性能基准测试指南

::: tip 文档信息
- **版本**: v4.71.7
- **日期**: 2026-06-02
- **项目版本**: v4.71.7
:::

## 概述

本指南详细说明如何进行性能基准测试、性能指标监控、性能优化评估等重要内容。

---

## 一、性能测试分类

### 1.1 负载测试

模拟正常用户访问，验证系统在预期负载下的表现。

### 1.2 压力测试

模拟超过预期的负载，验证系统的瓶颈和极限。

### 1.3 稳定性测试

长时间运行测试，验证系统在持续负载下的稳定性。

### 1.4 基准测试

基于已有基准进行对比测试，验证优化效果。

---

## 二、关键性能指标

### 2.1 API性能指标

| 指标 | 目标值 | 说明 |
|------|--------|------|
| 响应时间 (P95) | < 500ms | 95%请求的响应时间 |
| 响应时间 (P99) | < 1000ms | 99%请求的响应时间 |
| 吞吐量 | > 500 req/s | 每秒请求数 |
| 错误率 | < 0.1% | 错误请求占比 |

### 2.2 数据库性能指标

| 指标 | 目标值 |
|------|--------|
| 查询响应时间 (P95) | < 100ms |
| 连接池利用率 | < 80% |
| 慢查询数量 | < 10/hour |

### 2.3 前端性能指标

| 指标 | 目标值 |
|------|--------|
| LCP (最大内容绘制) | < 2.5s |
| FID (首次输入延迟) | < 100ms |
| CLS (累计布局偏移) | < 0.1 |
| TTI (可交互时间) | < 3.8s |

## 三、性能测试工具

### 3.1 API测试工具

#### Apache Bench (ab)

```bash
# 简单基准测试
ab -n 1000 -c 10 http://localhost:3001/api/health

# 带认证的测试
ab -n 1000 -c 10 -H "Authorization: Bearer token" http://localhost:3001/api/farm
```

#### k6

```javascript
// k6 测试脚本
import http from 'k6/http';

export const options = {
  vus: 50,
  duration: '30s'
};

export default function() {
  http.get('http://localhost:3001/api/health');
}
```

### 3.2 前端性能测试

#### Lighthouse

```bash
# 运行Lighthouse审计
lighthouse http://localhost:5173 --view
```

#### Web Vitals

```javascript
// 在前端代码中监控性能指标
import { getLCP, getFID, getCLS } from 'web-vitals';

getLCP(console.log);
getFID(console.log);
getCLS(console.log);
```

## 四、性能基准测试流程

### 4.1 测试环境准备

1. **隔离环境** - 使用独立的测试环境
2. **配置一致** - 与生产环境配置一致
3. **清除缓存** - 测试前清除缓存
4. **系统预热** - 先运行预热请求

### 4.2 基准测试步骤

1. **建立基准线** - 记录当前性能
2. **实施优化** - 应用性能优化
3. **对比测试** - 再次运行相同测试
4. **分析差异** - 对比性能变化

### 4.3 性能报告模板

```markdown
# 性能测试报告

- **测试日期**: 2026-05-21
- **测试环境**: Staging
- **项目版本**: v4.50.0

## 测试结果

| 指标 | 优化前 | 优化后 | 提升 |
|------|--------|--------|------|
| API响应时间(P95) | 650ms | 420ms | +35% |
| 吞吐量 | 320 req/s | 510 req/s | +59% |

## 结论

优化效果显著，符合预期目标。
```

## 五、性能优化建议

### 5.1 API优化

- 数据库查询优化
- 添加适当的缓存
- API响应压缩
- 合理的分页

### 5.2 前端优化

- 路由懒加载
- 组件按需加载
- 图片资源优化
- 虚拟滚动

### 5.3 数据库优化

- 索引优化
- 查询重写
- 连接池配置
- 读写分离
